# Proposal: Route Page Flow Redesign

## Why

The current route experience has the right technical capabilities, but the page still feels like a feature/debug surface. Assessment, map navigation, GPS recording, map diagnostics, plan, and gear compete for attention.

## What Changes

- Make route workspace a clean route-preparation entry point.
- Make the assessment tab a decision screen instead of another map surface.
- Keep the route tab as the map-first field cockpit.
- Move diagnostics and implementation readiness behind secondary disclosure.
- Present plan checkpoints as a timeline.

## Out Of Scope

- Full turn-by-turn navigation.
- Automatic rerouting.
- Backend, cloud sync, community routes, or rescue dispatch.
- Marketplace or affiliate gear recommendations.
