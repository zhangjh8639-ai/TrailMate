# TrailMate 路线详情四 Tab 重设计规格

日期：2026-06-20
状态：设计规格，待实现

## 背景

路线详情页已经具备评估、轻导航、计划、装备四个 tab，但仍有两个问题：

- 视觉上还像功能集合，缺少生产级 App 的清晰主次。
- 路线 tab 内的高德 SDK、授权、定位、通知、离线、轨迹记录交互混在一起，用户很难判断下一步。

本规格在现有能力基础上重新定义四个 tab 的职责和轻导航状态机。

## 产品原则

1. 一个 tab 只回答一个主问题。
2. 首屏不显示调试、SDK、Key、package、SHA1、AI prompt、历史证据。
3. 高德隐私同意前置到首次地图能力准备；Android 系统权限按功能触发。
4. 轻导航首屏只保留地图、当前位置、下一检查点、进度和主动作。
5. 不承诺救援、安全保证、自动重规划或专业导航替代。

## 授权与准备流

### 路线能力准备

新增轻量准备 flow，可在 onboarding 后或用户首次进入路线详情前触发。

用户看到：

- TrailMate 会使用高德在线底图；不同意时仍可使用本地 GPX 预览。
- TrailMate 会在用户点击 `开始徒步` 或 `使用当前位置` 时请求定位，用于当前位置、路线匹配和轨迹记录。
- TrailMate 会在开始记录时请求通知，用于切后台持续记录状态。

用户操作：

- `同意地图服务并继续`
- `稍后，仅使用本地路线`

系统权限：

- 定位权限只在用户点击 `开始徒步` 或 `使用当前位置` 时请求。
- 通知权限只在用户点击 `开始记录` 且系统需要时请求。

## 四 Tab 重新定义

### 评估

目标：回答“这条路线是否适合我”。

首屏结构：

- 适配结论：推荐 / 谨慎 / 不建议。
- 置信度：高 / 中 / 低。
- 三个路线指标：距离、爬升、预计用时。
- 关键风险 chips：天气、暴露、距离、爬升、装备、信号。
- 主动作根据结论变化：
  - 推荐：`进入路线`
  - 谨慎：`检查装备`
  - 不建议：`查看替代建议` 或 `查看计划`

二级内容：

- 为什么这样判断。
- 使用的数据类型说明，但不展示原始历史证据。

### 路线

目标：回答“我在哪、下一步去哪、现在按哪个按钮”。

首屏结构：

- 地图主体：GPX 路线、检查点、当前位置、精度圈、已记录轨迹。
- 顶部轻状态：地图源、本地/高德、定位状态。
- 底部导航控制台：
  - 当前检查点。
  - 下一检查点。
  - 完成进度。
  - 路线匹配状态。
  - 一个主按钮。
- 快捷工具：
  - 安全分享。
  - 标记点。
  - 离线/图层。
  - 结束。

主按钮状态：

- 未授权定位：`开始徒步`
- 已定位未开始：`开始徒步`
- 记录中：`暂停`
- 暂停中：`继续`
- 偏离路线：`查看恢复建议`
- 完成：`查看轨迹`

地图/高德策略：

- 高德同意、Key、SDK 都可用：使用高德在线底图。
- 高德未同意或不可用：使用本地 GPX 预览，不把问题渲染成错误首屏。
- 普通用户只看到“在线底图可用 / 当前使用本地路线 / 地图服务需同意后启用”。
- 开发诊断信息移动到设置或开发者详情，不在路线 tab 默认展示。

### 计划

目标：回答“何时补给、休息、核对风险”。

首屏结构：

- 今日行程计划：预计总时长、出发建议、最晚折返点。
- 准备状态：离线计划、补给点、休息点、天气复核。
- 检查点时间线：每个点展示里程、预计用时、行动建议、风险 badge。

文案要求：

- 没有真实天气源时，不写“天气稳定”；写“天气待复核”。
- 使用“建议”“复核”“参考”，避免确定性安全判断。

### 装备

目标：回答“这次要带什么，已有装备是否匹配”。

首屏结构：

- 路线清单 / 我的装备 / 详情 分段。
- AI 建议提示条，作为辅助而不是标题主角。
- 必带装备列表，按重要性排序。
- 每项显示状态、类别、匹配品牌型号、风险原因和动作。

装备项状态：

- 已匹配。
- 需检查。
- 缺失。
- 可选。

交互：

- `查看匹配`
- `添加已有装备`
- 添加装备使用底部 sheet，不打断路线清单上下文。

## 需要修改的现有位置

- `RouteDetailScreen.kt`
  - 拆分 route cockpit 组合函数。
  - 删除普通用户默认可见的 AMap launch diagnostics。
  - 将 `RouteCockpitDiagnosticsDisclosure` 改成“地图与定位帮助”，只显示用户语言。
- `RouteCockpitTab.kt`
  - 保持轻 wrapper，但传入更聚合的 presentation state。
- `RouteGearTab.kt`
  - 从 panel list 改成 route gear checklist。
- `RoutePlanTab.kt`
  - 增加行程节奏和天气复核语言。
- Onboarding / Profile
  - 新增地图服务 consent 入口或首次使用准备 flow。

## 验证标准

- OpenSpec 校验通过。
- 单元测试覆盖路线 tab 主按钮状态机。
- Compose UI 测试覆盖：
  - 评估 tab 不出现历史证据和 debug 文案。
  - 路线 tab 首屏不出现 Key、SDK、package、SHA1、diagnostics。
  - 高德未授权时路线 tab 使用用户语言提示，并可继续本地路线。
  - 计划 tab 使用“天气待复核”。
  - 装备 tab 有路线清单、我的装备、详情三个区域。
- 模拟器截图验证四个 tab 首屏风格统一。
