# TrailMate 路线驾驶舱设计规格

日期：2026-06-19
状态：草案，待用户确认后实现

## 目标

把“路线 / 轻导航”从信息堆叠页面改成真实徒步现场可用的路线驾驶舱。用户进入路线页后，应先获得现场行动判断，而不是看到历史证据、注册画像、调试诊断或 AI 输入材料。

成功标准：

- 首屏能回答“我在哪、下一段去哪、是否适合继续、现在按哪个按钮”。
- 注册画像、历史运动、身高体重、装备详情作为内部证据，不在主流程显性展示。
- 高德地图、真实 GPS 和轨迹记录成为导航页核心体验。
- 装备建议仍能回到“我的装备”，但路线驾驶舱只展示路线相关状态。

## 用户动线

1. 用户登录后完成一次轻量画像采集。
2. 用户在首页导入 GPX / KML，或选择历史路线。
3. App 用路线、天气、用户画像、历史运动和装备库存生成本地评估。
4. 用户进入路线详情，默认先看评估。
5. 用户切换到“路线”，看到地图驾驶舱。
6. 出发前，用户检查 GPS、记录、离线地图、装备四个状态。
7. 用户点击“开始徒步”，App 开始真实 GPS 轨迹记录。
8. 徒步中，App 展示当前位置、下一检查点、路线进度和偏航提醒。
9. 结束后，用户进入轨迹回顾和数据页。

## 页面结构

### 路线详情顶区

- 返回、首页、更多。
- 路线名，例如“龙井山脊”。
- 距离和累计爬升。
- 分段控制：评估 / 路线 / 计划 / 装备。

该区域保持轻量，不展示调试或证据来源。

### 路线驾驶舱

地图是主体，应在“路线”Tab 首屏占 58% 到 65% 的高度。

地图内容：

- GPX 参考路线。
- 检查点。
- 当前位置和 GPS 精度圈。
- 已记录轨迹。
- 起点、终点和当前检查点。

地图浮层：

- 当前检查点：名称、已完成距离、海拔。
- 下一检查点：距离、预计时间或爬升提示。
- 路线匹配：正常、偏离、GPS 较弱。
- 总进度：距离和爬升。

### 动作抽屉

动作抽屉固定在地图下方或底部区域，根据状态切换主按钮。

状态规则：

- 未授权定位：主按钮为“启用定位”。
- 已定位但未开始：主按钮为“开始徒步”。
- 记录中：主按钮为“暂停”，辅助按钮为“标记检查点”“结束”。
- 暂停中：主按钮为“继续”，辅助按钮为“结束”。
- 偏航：主按钮为“查看恢复建议”，辅助按钮保留“安全分享”。
- 已完成：主按钮为“查看轨迹回顾”。

### 现场检查条

现场检查条展示四个状态：

- GPS：可用 / 较弱 / 未授权。
- 记录：未开始 / 记录中 / 暂停。
- 离线地图：已准备 / 建议下载 / 不可用。
- 装备：通过 / 缺少关键项 / 建议检查。

点击任一项后，在地图下方展开详情。高德 key、隐私合规、定位服务和 SDK 诊断放在详情里，不占首屏。

### 计划与装备关系

计划 Tab 展示补给、休息和时间分配。页面文案应表达为建议，不表达为安全保证。

装备 Tab 展示本路线装备清单与“我的装备”匹配。用户可以从清单项进入添加品牌装备。装备详情不挤入路线驾驶舱首屏。

## 视觉方向

参考用户提供的三张截图：

- 背景：象牙白、雾白、浅绿地形底。
- 主色：深苔绿，用于主按钮和当前状态。
- 风险色：克制的琥珀 / 橙色，仅用于谨慎尝试、GPS 弱、装备缺失。
- 定位色：蓝色，用于当前位置、GPS 点、路线进度。
- 卡片：半径控制在 8 到 16 之间，避免页面套卡片。
- 图标：使用清晰线性图标，按钮里优先图标加短文字。

设计禁区：

- 不把所有模块堆在一个滚动页首屏。
- 不在真实页面展示“历史证据包”“用户画像证据”“AI prompt”。
- 不用大面积单一深色背景。
- 不用调试面板占据核心区域。

## 技术边界

首版不新增算法承诺，优先重组现有能力：

- 继续使用 GPX 几何、检查点、路线评估、装备建议、位置会话和轨迹记录引擎。
- 允许新增小型 presentation model，用于把现有状态合成为驾驶舱 UI 状态。
- 继续使用高德 Android 地图作为底图与 overlay 容器。
- 不做完整 turn-by-turn。
- 不自动重规划 GPX 路线。
- 不默认上传轨迹到云端纠偏。
- 不宣称替代专业地图、路标或救援工具。

## 验证

实现后至少验证：

- 单元测试：驾驶舱状态映射，覆盖未授权、未开始、记录中、暂停、偏航、完成。
- Compose UI 冒烟：路线 Tab 首屏存在地图、当前检查点、主动作、现场检查条。
- Android 构建：`./gradlew :android-app:assembleDebug`。
- 连接设备冒烟：定位权限、真实 GPS 记录和路线 Tab 首屏。

## 待产品确认

- 安全分享是否进入首版主动作抽屉。
- 附近补给点是否在首版使用高德 POI，还是保留为后续联网增强。
- 离线地图下载入口是否在首版放入现场检查条。
