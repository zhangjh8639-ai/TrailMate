# TrailMate 路线页动线重设计规格

日期：2026-06-20

## 目标

把路线页从“能力展示页”收敛成生产级徒步路线流程。用户应能按顺序完成：

1. 导入路线。
2. 判断是否适合。
3. 补齐装备和离线准备。
4. 开启真实 GPS 轻导航和轨迹记录。
5. 结束后进入复盘。

## 信息架构

### 路线 Tab

路线 Tab 只回答“当前目标路线是什么，下一步怎么进入准备”。它不展示完整评估、不展示现场 GPS 控制、不展示地图诊断。

首屏包含：

- 当前路线卡。
- 导入或更换 GPX。
- 路线状态摘要：评估、装备、离线、导航。
- 继续准备主按钮。

### 评估 Tab

评估 Tab 只回答“这条线适不适合我”。它不再复用大地图。

首屏包含：

- 匹配结论。
- 距离、爬升、预计用时。
- 最多三个风险因素。
- 可信度。
- 一个主动作：检查装备或进入路线。

### 路线 Tab 内的驾驶舱

路线详情里的“路线”Tab 回答“我现在在哪里，下一步怎么走”。

首屏包含：

- 地图主体。
- 当前检查点。
- 下一检查点。
- 进度。
- GPS、记录、离线、装备四状态条。
- 一个主动作：进入全屏导航或修复出发前阻断项。

普通路线页的展开详情只承载检查点、补给、定位可靠性、离线底图和真机排障证据；`开始徒步`、暂停/继续、标记检查点、结束记录等行进控制只出现在全屏导航中。

### 计划 Tab

计划 Tab 展示补给、休息、风险核对的时间线。文案使用建议语气，不做安全承诺。

### 装备 Tab

装备 Tab 只展示当前路线装备准备，不展示商品化推荐。

## 视觉方向

- 沿用象牙白背景、深苔绿主色、蓝色定位、琥珀风险色。
- 评估页像行前卡片，不像地图页。
- 现场地图可以有浮层，但浮层必须服务“下一步动作”，不服务工程解释。
- 调试和诊断只放在“地图与定位设置”折叠区。

## 验收

- 路线 Tab 未打开详情前，不出现路线评估、现场状态、保存路线包或路线驾驶舱。
- 进入路线详情默认看到评估卡，而不是地图。
- 评估页可一屏看清结论、风险、预计用时和下一步。
- 路线驾驶舱首屏有地图、当前检查点、主动作、状态条。
- 计划页是时间线，不是普通面板列表。
- OpenSpec 能验证新增路线流程要求。
