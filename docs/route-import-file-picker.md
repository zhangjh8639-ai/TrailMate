# 路线导入文件选择器

本切片把 `路线` Tab 的 GPX/KML 导入从静态样例推进到真实 Android 文件选择：

- 点击 `导入 GPX / KML` 会打开系统文件选择器；
- 选择文件后通过 `ContentResolver` 读取 URI；
- 读取结果交给现有 `RouteImportParser` 解析；
- 路线页展示真实文件名、路线名、距离、爬升、航点数、轨迹点数、海拔数据和质量提示；
- 取消、读取失败、格式不支持、XML 解析失败和缺少可导航轨迹都会显示明确中文状态；
- 成功解析只生成本次导入预览，不会静默保存到路线资产；
- 用户点击 `保存到路线` 后，会先写入本机 SQLite，再在路线资产列表顶部生成一张导入路线卡片，并标注 `本次已加入路线列表`。

## 保存行为

`保存到路线` 只在解析成功预览中可用。保存后的导入路线卡片会使用克制的路线资产文案：

- 来源：`GPX 导入` 或 `KML 导入`；
- 区域：`导入路线`；
- 离线状态：`仅轨迹可用`；
- 预计耗时：`待确认`；
- 难度：`未验证`；
- 可信度：`可信度待确认`；
- 标签：`导入轨迹`、`未验证`、`不含地图底图`。

核心模型语义上，导入路线保持 `Private`、`TrackOnly`、`Unverified`。这避免把用户自己导入的 GPX/KML 轨迹误呈现为平台认证路线，也不会暗示包含商业地图底图。

同一个导入预览重复点击保存会按导入 identity 更新本机记录，不会生成重复卡片。App 进程重启后会从本机 SQLite 恢复已保存的导入路线卡片。

保存后的导入卡片不展示 `开始导航` 或 `查看详情` 操作；这两个入口会在后续真实路线详情和轨迹导航切片中接入。

## 边界

本切片不做：

- 保存到 Room/DataStore 或服务端；
- 路线详情页跳转；
- 真实开始轨迹导航；
- MapLibre 地图渲染；
- GPS / Foreground Service / 后台轨迹记录；
- 离线地图包下载；
- 规划、装备、社区、商城或复杂出发前检查。

预览卡中的 `查看详情`、`开始轨迹导航` 仍是下一步 affordance，不产生详情页跳转或导航副作用。保存后的导入资产卡片不会展示这些未接入的动作。失败状态只提供重新选择文件。

## 文件与隐私

导入入口使用 Android 系统 document picker，不申请 `READ_EXTERNAL_STORAGE`、`WRITE_EXTERNAL_STORAGE`、`MANAGE_EXTERNAL_STORAGE` 或 `READ_MEDIA_*` 这类广泛存储权限。

导入文件只包含路线轨迹和航点，用于轨迹导航、偏航判断和进度计算，不包含商业地图底图。

本机 SQLite 只保存导入路线的元数据和有序轨迹点：文件名、来源格式、路线名、距离、爬升、航点数、轨迹点数、是否包含海拔、导入时间、隐私/可信边界和坐标序列。它不会保存商业地图瓦片，也不会把导入轨迹上传或公开。

## 当前保护

- 支持常见 GPX/KML 扩展和 MIME 类型；
- 对通用 MIME 类型保留扩展名 fallback；
- 对无扩展名但明确为 GPX/KML 的 MIME 类型补内部解析扩展名；
- 单文件读取上限为 2MB；
- 大文件显示 `文件过大，暂不支持直接导入`；
- 读取异常显示 `无法读取所选文件`；
- parser 已拒绝 `DOCTYPE/ENTITY`，避免外部实体风险。

## 手动验证

1. 构建并安装 debug APK。
2. 打开 TrailMate，切到 `路线` Tab。
3. 点击 `导入 GPX / KML`。
4. 选择一个本机 GPX/KML 文件。
5. 确认路线页显示真实文件名和解析指标。
6. 点击 `保存到路线`，确认路线资产列表顶部出现导入路线卡片，且显示 `GPX 导入`/`KML 导入`、`仅轨迹可用`、`待确认`、`未验证`、`不含地图底图` 和 `本次已加入路线列表`。
7. 再次点击 `保存到路线`，确认不会出现重复导入卡片。
8. 强制停止并重新打开 App，进入 `路线` Tab，确认已保存导入路线仍在列表顶部，且文案仍为 `仅轨迹可用`、`未验证`、`不含地图底图`。
9. 再选择无效文件或取消选择，确认不会显示伪造成功结果，也不会创建导入路线卡片。

## 已知限制

- 当前使用轻量 `SQLiteOpenHelper` 包装本地 SQLite；后续完整本地数据层可以迁移到 Room/Repository/Hilt。
- 当前解析成功预览中的详情和开始导航按钮不执行副作用；下一步应拆分为路线详情和轨迹导航入口两个独立切片。
- 当前持久化路线主轨迹点和导入指标；结构化航点、风险点和撤退点的完整持久化会在路线详情/导航切片中继续扩展。
